# 验证报告模板

生成模块或阶段验证报告时使用此模板。

## 命名规则

报告保存目录：

```text
reports/verification/
```

文件名格式：

```text
YYYYMMDD-HHMM-<阶段ID>-<模块ID>-verification.md
```

示例：

```text
20260702-1530-stage-01-input-loader-verification.md
```

## 状态值

- `PASS`：证据充分，可以提交给用户确认。
- `WARN`：核心功能可用，但有风险或异常样本，需要用户决策。
- `FAIL`：模块不能验收，必须修复后重新验证。
- `BLOCKED`：缺少用户输入或外部条件，无法继续验证。

## 报告完成检查清单

- 包含完整可复现命令。
- 包含输入和输出路径。
- 包含数量、指标和样本。
- 明确标记异常样本。
- 包含 provider 模式和降级行为。
- 明确说明用户是否可以确认通过，或是否需要先做决策。
